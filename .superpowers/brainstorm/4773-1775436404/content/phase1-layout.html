Phase 1 — Egg Layout Question

The spec says all 3 eggs are visible simultaneously, each with their own block cluster. But on mobile portrait (the primary target), this creates a real tension: the player is supposed to strategically aim for one specific egg — but Breakout ball control is famously imprecise. Which layout approach makes more sense?

A

All 3 Eggs Visible Simultaneously (Original Spec)

3 eggs + block clusters fill the upper screen. Player uses a single paddle across the full width. "Aiming" means nudging ball left/right to favor one egg's cluster. First egg fully hatched wins — others fade.

Keeps

  • Classic Breakout feel
  • Visual choice is immediate — you see all 3 paths

Risk

  • Mobile screen is narrow — 3 eggs + blocks = tiny hitboxes
  • "Aiming" in Breakout is indirect, so choice feels random
B

Choose First, Then Play

Before the Breakout game starts, player taps the egg they want to hatch. The chosen egg fills the screen with its unique block layout. The other two eggs are shown as small locked icons. Game plays as a focused single-egg Breakout.

Keeps

  • Explicit meaningful choice
  • Full mobile screen for Breakout — bigger, more playable
  • Each egg can have a unique, distinctive block pattern

Risk

  • Choice is divorced from gameplay — less emergent
  • Loses the "race" tension between eggs
C

Sequential Swipe — One Egg at a Time, Race Logic

Only one egg + its blocks fill the screen. Player swipes left/right to switch which egg is "active." Progress on each egg is tracked independently. First egg to reach 100% breaks and hatches.

Keeps

  • Full mobile screen per egg — large, readable
  • Strategic switching adds depth (work on easier egg vs. risk-reward)
  • Race tension preserved — all 3 are in play

Risk

  • More complex UX to explain
  • Swipe-to-switch conflicts with swipe-to-aim
